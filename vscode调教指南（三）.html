vscode调教指南（三）

发表于 2018-10-28 | 分类于 IDE

vs code调教系列最后一篇来介绍一下一些vscode的常用配置和使用技巧。

常用设置

设置简体中文

vs code默认是英文的，如果要将语言改成其它的语言，要先到插件市场下载其它语言的语言包，比如简体中文就去下载Chinese (Simplified) Language Pack for Visual Studio Code语言包。安装语言包后默认是不启用的。需要先通过ctrl + shift + p打开命令面板，搜索config找到Config Display Language，然后点击打开locale.json，设置"locale": "zh-cn"，最后重启就可以看到vs code的已经修改语言为简体中文。

优秀的开发者是不用中文的。

自动保存

每次都ctrl + s来保存文件我是觉得很麻烦，所以我改成了隔一段时间自动保存的那种模式。从上面的vscode设置说明可以看到，vs code支持4种关于自动保存的模式，依次是关闭自动保存，定时保存，编辑区失去焦点时自动保存，关闭窗口时自动保存。通过"files.autoSaveDelay": 1000设置间隔时间，这个时间不要设置太短，建议大于1500毫秒，太短容易出现打字移位的bug。

启用主题和文件图标

主题和文件图标从扩展市场下载安装后默认是不会立即生效的，需要在设置中或者通过ctrl + k ctrl + t调出设置主题面板来设置启用哪一个主题和图标。

修改默认的shell

默认的powerShell不好用，而且没有着色。可以通过下面的设置把默认的powershell换成你想设置的shell，比如git bash，cygwin。

设置文件关联

举个例子，微信小程序官方MINA框架使用的样式文件后缀名是.wxss，其实写起来就是css的语法，但是vs code他是根据文件的后缀名来启用插件和语法高亮着色的，vs code是不认.wxss的。

我们可以通过文件关联让vs code将.wxss后缀名的文件当做.css文件来处理。

关闭自动猜测文件编码

这个功能开启了后我发现经常性猜不准，明明是utf-8的编码老猜成gbk的编码(￢︿̫̿￢☆)。

建议大家写代码用utf-8编码，利人利己。

技巧篇

搜索插件

vs code越来越流行，nb的插件层出不穷，那我怎样让你自己紧跟社区的脚步一直使用最前沿最流行的插件以及找到适合自己的项目的插件呢？

可以常去vs code扩展市场官网去看看。

往往我们在用不同的技术栈时需要对应的插件，尤其是代码片段（snippets）和语法高亮以及语法支持。

比如当我们使用vue来写前端项目需要代码补全来提高效率就可以在插件市场搜索vue或者vue snippets。

代码片段提供的代码补全可以大大减少我们写重复代码的时间，所以每次我开始新的项目时都会去插件市场搜一下有没有对应技术的代码片段插件。

除了代码片段插件，语法高亮插件也是非常必要的。针对这一类插件可以模板名比如（numjucks），后缀名比如(.conf)，也可以搜索文件名（nginx.conf）。

语法支持类的插件比如Markdown All in One提供了markdown的语法检测以及风格检测，让你的代码更加规范补不容易出错。

快速编写css

对于一些属性的名称较短的，例如：display 与 visibility，输入属性首字母与值的首字母即可。比如：df 是 display: block; ，dn 是 display: none;。

对于一些属性的值是数值，例如：padding，margin，left，width 等，输入属性首字母与值即可。比如，m1 是 margin: 1px;。单位默认是 px，不过你可以指定一下单位，比如：w2vw 就是width: 2vw;。当值是百分比时比较特殊，字母 p 代表 %，比如：w5p 就是 width: 5%;。

名称较长的属性往往含有连字符（-），输入连字符前后两个单词的首字母再加上值即可。比如：pt10 是 padding-top: 10px;。

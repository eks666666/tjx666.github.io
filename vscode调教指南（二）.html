vscode调教指南（二）

发表于 2018-10-27 | 分类于 IDE

因为我经常写前端这块，所以我安装了很多写前端非常实用的插件。下面来一一介绍它们。

Auto Close Tag

功能就像名字一样，这个插件辅助标签语言自动闭合标签，并且在闭合标签后将光标定位到标签中间。这个插件还能在你插入标签以后从1.16版本开始内置的自动闭合html标签功能，这个插件主要支持在XML，PHP， Vue， JavaScript， TypeScript， JSX， TSX 等文件类型中也能自动闭合标签。

Auto Rename Tag

这个插件会在你修改一对闭合标签中的一个时，同步修改另一个标签。

Beautify

一个代码格式化插件，虽然vscode内置对很多文件的格式化支持，但是往往有地方并不是你希望的格式。这个插件通过编写.jsbeautifyrc可以让你按照你想要的格式格式化你的代码。

Bootstrap 4, Font awesome 4, Font Awesome 5 Free & Pro snippets

提供bootstrap4代码补全,当然也有bootstrap3的代码片段插件。

Class autocomplete for HTML

自动补全class后面的=""，并将光标定位到引号中间。这个插件会根据当前打开的html文件引用样式表链接扫描对应的css文件，来提供类名的自动补全。每次打开或修改对应的css文件都会更新提供自动补全的类名。

CSS Peek

提供根据类名和id选择器跳转到其定义处（ctrl + 左击）或显示样式内容（alt + f12）。

ES6 Mocha Snippets

提供mocha测试代码片段。

ES7 React/Redux/GraphQL/React-Native snippets

提供ES7 React/Redux/GraphQL/React-Native代码片段。

escaping-characters

用来将字符转换成unicode编码。

ESLint

eslint是目前前端最流行的代码风格检查工具。这个插件将eslint集成到了vscode中，如果当前打开的页面中有代码风格问题，会在problems面板中有提示。很多命令都通过ctrl + shift + p调出命令面板搜索到。更多信息可以查看官方文档。

HTML CSS Support

这个插件比之前介绍的Class autocomplete for HTML更厉害，它会直接扫描当前工作区的的css和scss文件，提供类名和id补全,甚至网络上的css文件（通过链接）。

HTML Snippets

这个插件提供完整的html5标签，html着色，代码片段，对于已经被放弃的标签还会给出提示。

Image preview

在编辑区左侧显示图片的缩略图。如果鼠标悬浮到图片路径或链接上会悬浮显示图片。

Import Cost

显示使用require或import方式导入的js模块的文件大小。

IntelliSense for CSS class names in HTML

功能和Class autocomplete for HTML以及HTML CSS Support类似。都是提供类名代码补全的。

JavaScript (ES6) code snippets

提供ES6代码片段。

Live Server

实现html自动刷新。

Node.js Assertion Snippets

提供nodejs断言代码片段。

npm Intellisense

在npm模块导入语句中提供自动补全。

Regex Previewer

提供正则匹配预览，可以左边写正则，右侧写需要匹配的字符串，匹配成功的字符串会被高亮。

Search node_modules

有时候需要看node模块里面的readme，这个插件方便你通过命令面板找到node包。

SVG Viewer

vscode默认不支持预览svg文件。这个插件让vscode可以直接预览svg图像。

open in browser

使vscode支持在浏览器打开工作区html文件。

Webpack Snippets

提供webpack代码片段。

vscode wxml

为 VSCode 提供 wxml 语法支持及代码片段。

wechat-snippet

微信小程序代码辅助,代码片段自动完成。

eggjs

提供eggjs的代码片段，调试和配置检查功能。具体参见详情页。

(完)

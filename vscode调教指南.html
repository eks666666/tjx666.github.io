vscode调教指南

发表于 2018-10-25 | 分类于 IDE

从小白到如今的老司机，先后折腾过不少编辑器和IDE。我也曾经有过编辑器和IDE傻傻分不清的时候。现在主要使用Jetbrains全家桶和vscode，偶尔修改linux系统配置的时候用下vim。微软公司开源的vscode会像如今这么受欢迎，我认为主要有几个优点：

  1. 强大又灵活的插件系统
  2. 高度可定制化
  3. 开源免费
  4. 跨平台
  5. 轻量，启动速度快

通过合理的安装配置插件，完全可以将vscode编辑器打造成大大提高编码效率的IDE。这篇文章我将介绍如何“调教”vscode，让它看起来更美观舒服，集成各种实用功能，大大提高编码效率。搜人鱼，不如授人以渔。我不希望只是教会读者记住一堆过段时间就忘了的名词，我更希望教会大家如何自己去发现和配置。

主题(theme)

通过ctrl + k ctrl + t可以调出vscode的主题设置。vscode的快捷键很有规律。有相当一部分是以ctrl + k为前缀的。作为一个开发者，不一定要求英语特别好，但是常见的单词的还是要会的，更重要的是: 要学会猜单词。调出主题设置的这个快捷键中的t猜测应该是theme的意思。

对于主题，不同的人有不同的喜好。有些人喜欢暗色调（dark），有些人喜欢浅色调（light）。我个人比较喜欢暗色调，觉得看着不容易累。使用快捷键ctrl + shift + x打开插件（extensions）市场。搜索theme，点击右上角的三个点打开过滤选项，选择安照安装数量(install count)过滤搜索结果，这样我们就得到了当前主题的安装数量排行榜。

截止至本博文创作时，根据安装数排名前5的vscode主题应该是:

  1. One Dark Pro
  2. Material Theme
  3. Dracula Official
  4. Atom One Dark Theme
  5. Winter is Coming Theme

这里面很多主题笔者都尝试过，隔一段时间审美疲劳了就换一个。我比较喜欢的主题有：

深色：

  1. An Old Hope
  2. One Dark Pro
  3. One Monokai Theme
  4. Dracula Official

浅色：

  1. Solarized Light vscode 内置的 浅色主题。米黄色背景，看起来很舒服
  2. GitHub Plus 给你github的风格体验

读者可以自行安装体验。

字体(font)

默认的调出预设的快捷键是ctrl + ,，但是一般会和系统快捷键冲突，所以被我改成了ctrl + alt + s，关于怎样设置快捷键，后面再叙述。

字体必须系统安装了你的设置才会生效，很多字体都可以再github找得到。

我的字体设置：

在vscode中关于字体的设置分为编辑区（editor），终端（terminal）。终端的字体默认采用编辑区的设置，当前面的字体不支持时采用后面设置的字体。

justjavac对于什么样的字体适合lu代码说过：

非常清晰（Crisp clear characters）

支持扩展字符集（Extended characterset）否则会出现乱码或者方格

对空白把握的很好（包括但不限于空格、非中断空格、制表符…）

可轻易区分’l’, ‘1′ 和 ‘i’

可轻易区分’0′, ‘o’ 和 ‘O’

可轻易区分左右引号 - 最好是对称的

清晰的标点符号，特别是大括号小括号和中括号

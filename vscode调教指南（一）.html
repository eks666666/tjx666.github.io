vscode调教指南（一）

发表于 2018-10-25 | 分类于 IDE

从小白到如今的老司机，先后折腾过不少编辑器和IDE。我也曾经有过编辑器和IDE傻傻分不清的时候。现在主要使用Jetbrains全家桶和vscode，偶尔修改linux系统配置的时候用下vim。微软公司开源的vscode会像如今这么受欢迎，我认为主要有几个优点：

  1. 强大又灵活的插件系统
  2. 高度可定制化
  3. 开源免费
  4. 跨平台
  5. 轻量，启动速度快

通过合理的安装配置插件，完全可以将vscode编辑器打造成大大提高编码效率的IDE。这篇文章我将介绍如何“调教”vscode，让它看起来更美观舒服，集成各种实用功能，大大提高编码效率。授人鱼，不如授人以渔。我不希望只是教会读者记住一堆过段时间就忘了的名词，我更希望教会大家如何自己去发现和配置。

主题(theme)

通过ctrl + k ctrl + t可以调出vscode的主题设置。vscode默认的快捷键很有规律。有相当一部分是以ctrl + k为前缀的。

作为一个开发者，不一定要求英语特别好，但是常见的单词的还是要会的，更重要的是: 要学会猜单词。调出主题设置的这个快捷键中的k应该是key表示，t应该是theme的意思。

对于主题，不同的人有不同的喜好。有些人喜欢暗色调（dark），有些人喜欢浅色调（light）。我个人比较喜欢暗色调，觉得看着不容易累。使用快捷键ctrl + shift + x打开插件（extensions）市场。搜索theme，点击右上角的三个点打开过滤选项，选择安照安装数量(install count)排序搜索结果，这样我们就得到了当前主题的安装数量排行榜。

截止至本博文创作时，根据安装数排名前5的vscode主题应该是:

  1. One Dark Pro
  2. Material Theme
  3. Dracula Official
  4. Atom One Dark Theme
  5. Winter is Coming Theme

这里面很多主题笔者都尝试过，隔一段时间审美疲劳了就换一个。我比较喜欢的主题有：

深色：

  1. An Old Hope 比较中意它的配色
  2. One Dark Pro atom的默认主题，添加了很多图标
  3. One Monokai Theme sublime的默认主题
  4. Dracula Official Intellij系列好像都内置Dracula主题

浅色：

  1. Solarized Light vscode 内置的 浅色主题。米黄色背景，看起来很舒服
  2. GitHub Plus 给你github的风格体验
  3. Tiny Light 仿HBuilder默认主题风格

读者可以自行安装体验。

快捷键(shortcuts)

前面说过，vscode的快捷键设置很有规律。一部分是以ctrl + k作为前缀，和光标(cursor)有关的大多都有alt，很多插件以及panel中有3个快捷键都是以ctrl + shift或者ctrl + alt作为前缀，还有一些快捷键是重复两遍输入，比如ctrl + alt + i ctrl + alt + i。

要记住这些快捷键，靠背是效率很低的，毕竟快捷键那么多。有人说了，记那么多快捷键干嘛，用鼠标点点不是挺好的嘛。记住ctrl + s，ctrl + z，ctrl + c, ctrl + v那几个不就行了，搞那么多花里花俏的。在我看来，熟练使用快捷键不仅仅能大大提高你的操作速度，还能减少你碰鼠标的次数，减少手臂的疲劳。

那该怎样记住那么多快捷键呢？其实没必要刻意去记。

需要知道某些功能的快捷键是什么或者自定义快捷键直接在快捷键设置界面搜索或设置就行了。

通过默认快捷键ctrl + k ctrl + s打开快捷键设置界面，这个s应该是shortcuts的意思。

查找快捷键

有时候我们可能会忘了某些快捷键，这个时候我们可以通过默认快捷键ctrl + k ctrl + s打开快捷键设置界面，搜索一些关键字通过模糊匹配找到对应的快捷键。

只要搜索的关键字比较准，就很容易找到自己想要设置的快捷键。平时稍加记忆一下那些常见的关键字的英文比如：bar（栏），menu（菜单），cursor（光标），move（移动），preview（之前），next（下一个），fold（折叠），indent（缩进），suggestion（提示）等等。举个例子：你要查找将当前选中的代码复制到上一行的快捷键，那你就可以搜preview，复制到下一行就可以搜next。

可以进快捷键搜索页面多看看，不认识的单词查一查，这样以后搜索的就会越来越准确。

如果时查找和插件相关的快捷键就直接搜索插件名或者进入插件详情页查看。

在vscode中搜索快捷键时，不单单可以根据command(命令)搜索，也可以直接搜索绑定的键位(KeyBinding)，比如ctrl alt：

当然根据Sourrce或者When搜索也是可以的。可见vscode做的还是很人性化的。

设置快捷键

设置快捷键只需要你先找到对应的快捷键，选中你要设置的快捷键，左侧有个🖊，点击后就开始设置快捷键。按照你要设置的快捷键的键位和顺序键入，如果和已经存在的快捷键冲突会有提示。回车表示确认设置，如果有冲突会覆盖之前的快捷键设置。

设置快捷键如果没什么特别好记的键位和键序，建议使用ctrl + k或者ctrl + alt开头。

下面列举一些非常实用的快捷键：

打开面板或者切换窗口类的：

  1. ctrl + shift + p 打开命令搜索。几乎所有的插件命令都可以通过命令搜索找到，点击搜索到的命令执行该命令

  2. ctrl + p 搜索文件。这个p应该是path的意思。当文件嵌套很深或者你不喜欢用鼠标点时，可以通过这个命令调出搜索文件框，快速打开文件

  3. ctrl + shift + b 打开或隐藏侧边栏。这个b可以理解为beside。有时候因为编辑区太窄了，需要隐藏侧边栏，这个时候就很方便了

  4. ctrl + shift + x 我经常有事没事去扩展市场看看有没有什么好玩的插件🙂

  5. ctrl + shift + e 打开文件列表。这个e就是explorer

  6. ctrl + shift + n 打开新窗口。n即new

  7. ctrl + alt + s 默认的时ctrl + ,因为和系统快捷键冲突被我改了

  8. ctrl + k ctrl + s 打开快捷键设置。这个s就是shortcuts嘛

  9. ctrl + k ctrl + o 打开文件夹。这个o应该是open的意思

  10. ctrl + k ctrl + q 回到编辑区最后编辑的地方

  11. ctrl + k ctrl + w 关闭当前编辑区所有打开的标签页，w即window

  12. ctrl + k z 切换zen mode，我要专心写代码了😄

    zen mode下全屏截图：

  13. ctrl + ` 切换终端(terminal)

  14. ctrl + shilt + m 切换问题面板

  15. ctrl + shift + y 切换 debug 控制台

  16. ctrl + shift + d 打开 debug 侧边栏

  17. ctrl + w 关闭当前焦点所在编辑区的标签页

  18. alt 切换菜单栏。我把菜单栏设置为toggle模式了，默认不显示，按下alt键显示

  19. ctrl + n 新建标签页，这个n也是new的意思喽

  20. ctrl + r 打开最近打开过的文件夹

  21. ctrl + alt + p 重新打开当前窗口，默认不是这个,可以搜索关键字reload。

编辑类的：

  1. ctrl + d 键入一次选中光标所在单词，键入两次选中当前光标所在词以及下一个和当前光标所在词相同的词，键入三下，选中所有和当前光标所在词相同的词
  2. ctrl + shift + k 删除当前行
  3. alt + shift + f 格式化代码。这个f应该是format的意思
  4. ctrl + space 触发代码提示
  5. ctrl + enter 无论当前光标处于当前行的什么位置，在当前行下一行插入一行
  6. ctrl + shift + enter 无论当前光标处于当前行的什么位置，在当前行上一行插入一行
  7. ctrl + alt + ↓ 复制当前选中部分到下一行（next line），如果没有选中就复制当前行
  8. ctrl + alt + ↑ 复制当前选中部分到上一行（preview line），如果没有选中就复制当前行
  9. ctrl + [ 将选中部分向左缩进，没有选中部分就缩进当前行
  10. ctrl + ] 将选中部分向右缩进，没有选中部分就缩进当前行
  11. ctrl + shift + [ 将当前光标所在代码块折叠（fold）
  12. ctrl + shift + [ 将当前光标所在的折叠代码块展开
  13. ctrl + k ctrl + 0 折叠所有代码块
  14. ctrl + k ctrl + j 展开所有代码块
  15. ctrl + k ctrl + level 折叠level级别代码块
  16. ctrl + i 选中当前行，重复x次，选中从当前行开始往后x行
  17. ctrl + home 跳转到编辑区最开头
  18. ctrl + end 跳转到编辑区最后
  19. ctrl + shift + l 选中编辑区内所有和当前选中文本一样的文本
  20. ctrl + f 在当前标签页查找。f 即find，可以先选中你要搜索的词再打开搜索，这样就不用手动输入搜索词了
  21. ctrl + h 在当前标签页查找并替换
  22. ctrl + g 跳转到某一行。g即go
  23. ctrl + shift + 0 将焦点转移到侧边栏
  24. ctrl + shift + f 当焦点在侧边栏时，打开在所有工作区搜索功能
  25. ctrl + shift + h 当焦点在侧边栏时，打开在所有工作区搜索替换功能
  26. ctrl + \ 拆分当前编辑区
  27. ctr + number 打开或选择number号编辑区
  28. f12 跳转到选中变量定义处
  29. ctrl + alt + d ctrl + alt + d 为当前光标所在函数或类添加文档注释，d应该是document的意思
  30. ctrl + k 清除控制台内容

其实vscode有很多编辑技巧。比如任意位置添加多个光标，多位置同时编辑，其实只需要按住alt键同时点击你需要添加光标的地方就可以了。又比如跳转变量定义处还可以通过按住ctrl键同时左击变量即可，如果控制台有文件名或者链接也可以通按住过ctrl同时左击它们打开对应文件或打开浏览器请求链接。更多技巧读者可以慢慢探索。

扩展(extensions)

扩展也叫插件，使用electron开发的vscode必然扩展性极强。可以说vscode能有如今这样受欢迎，其扩展的丰富和强大有很大的功劳。vscode插件使用typescript开发，非常易于编写，安装和使用也很方便，并且扩展的贡献者非常多和活跃，未来vscode肯定会继续出现各种有趣和强大的插件。

通过ctrl + shift + x默认快捷键打开扩展市场。

vscode提供很多的排序和过滤选项：

不像sublime，vscode下载插件很稳定，不需要FQ，下载时从来没碰到什么问题。

插件的安装数和星级都是很好的选择一个插件的指标。

点击上图侧边栏中的插件项就可以进入插件详情页面，插件的详情页一般会介绍插件的功能，使用方法，快捷键，怎样配置等。

内置插件

在插件搜索中使用Show Built-in Extensions过滤选项可以看到vscode还是内置了很多的插件的。大部分都是和前端相关的构建，打包，调试，还有语法检测插件。在这里我只提一下emmet插件。

emmet插件提供了emmet语法支持，方便前端更快速的写出html代码，在你按照emmet语法输入时，如果可以使用emmet插件导出html代码的话就会有代码提示，其实也可以通过有没有代码提示判断你emmet语法有没有错误，通过tab键或者enter回车就可以直接导出html代码。

通用插件

Better Comments

加强注释功能

插件名字翻译过来就是更好的注释，事实上使用这个插件确实能在注释方面带来很大的帮助。像下面这张官方图一样，在注释中如果在注释行头部加入!，*，?，TODO语法标记，参数或返回值，插件会给它们设置不同的样式。甚至可以通过配置指定标记和样式来达到自己的特殊需求。

Code Runner

快速运行代码

官方介绍：

Run code snippet or code file for multiple languages: C, C++, Java, JavaScript, PHP, Python, Perl, Perl 6, Ruby, Go, Lua, Groovy, PowerShell, BAT/CMD, BASH/SH, F# Script, F# (.NET Core), C# Script, C# (.NET Core), VBScript, TypeScript, CoffeeScript, Scala, Swift, Julia, Crystal, OCaml Script, R, AppleScript, Elixir, Visual Basic .NET, Clojure, Haxe, Objective-C, Rust, Racket, AutoHotkey, AutoIt, Kotlin, Dart, Free Pascal, Haskell, Nim, D, and custom command

可以看到支持直接运行代码片段或文件的编程语言非常多。安装后右上角会多出一个三角形的运行按钮（下面图中红色方框），这个也可以在设置中设置隐藏。

常规操作：alt + n清除output，ctrl + alt + n直接运行代码。

Code Spell Checker

手残党的救星

如果一个词在这个插件的词库中没有被添加，即认为你拼写错误，就会在那个被认为拼写错误的词下面添加绿色的波浪线。当然有可能插件识别为拼写错误的词完全是没有错误的比如：lyreal666😊。你可以选中那个词右击添加到插件词库中，这样就不会提示错误了。

这个插件还有智能提示的功能，选中被视为拼写错误的词，按快捷键ctrl + .就会列出词库中和这个词相近的词。

EditorConfig for VS Code

方便生成.editorconfig文件，以及提供editorconfig代码提示。

file-size

在右下角状态栏显示当前打开的文件大小。

gitignore

快速生成各种项目类型的.gitignore文件。

Guides

显示缩进线的一个插件。内置的和另一个安装数非多的显示缩进线的插件indent-rainbow我都不太喜欢。

Output Colorizer

给log文件语法着色。

Path Intellisense

提供文件路径自动补全。

Settings Sync

官方介绍：通过github gist跨多台设备同步设置，代码片段，主题，文件图标，启动设置，快捷键绑定，工作区和插件。

就是个备份还原vscode配置的工具。

这个插件的详情页有详细的教程讲怎样备份还原。

Terminal Here

有时候我们需要将当前工作路径切换到某个文件所在路径，通过这个插件只需要在文件列表侧边栏右击文件，选择open in terminal就行，非常的方便。当然，这个插件也提供了快捷键，不过需要你自己去设置，可以在快捷键设置界面搜索open in terminal。

TODO Highlight

高亮 TODO, FIXME 和其它在你代码里面的注解。

vscode-icons

给你的文件夹和文件添加图标。切换文件图标的方式和切换主题的方式一样。

翻译(translate to chinese)

想起我四级还没过，我要滚去背单词了😔。

选中需要翻译的词然后使用快捷键ctrl +shift + t右下角便会弹出翻译。

Paste JSON as Code

把jison代码当作其它语言拷贝，支持的语言包括TypeScript, Python, Go, Ruby, C#, Java, Swift, Rust, Kotlin, C++, Flow, Objective-C, JavaScript, Elm, and JSON Schema。

vscode-fileheader

在文件顶部添加注释标注，这个插件会在保存文件时自动更新最后修改时间。在插件详情页有详细的配置方法。

Git History

在标签页中显示提交历史。

YAML Support by Red Hat

提供yaml语法检测。

Color Highlight

高亮代码中的颜色值背景。

Emoji Code

提供在 HTML, CSS, JavaScript/CoffeeScript/TypeScript/LiveScript, JSON, C#, Laravel Blade, Markdown, Python and Ruby中插入编码后的表情代码。

Power Mode

提高vscode的B格。

Markdown All in One

像插件名称一样提供vscode对markdown的各种支持。更多信息查看详情页。

Markdown Preview Enhanced

这个markdown预览插件渲染效果不错。

markdownlint

markdown风格检测工具。

nginx.conf

提供nginx.config语法高亮。
